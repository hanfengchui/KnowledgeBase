# 面试演示脚本

## 1. 项目定位

这是一个“企业知识库 + 业务查询 AI 助手”。它不是简单调用大模型，而是把企业私有知识和业务系统能力接入大模型：

- 私有知识：通过 RAG 解决“模型不知道企业内部资料”的问题。
- 业务系统：通过 Tool Calling 解决“模型需要实时业务数据”的问题。
- 结构化输出：接口统一返回 answer、sources、stats、toolCalls，前端可观测。

## 2. RAG 链路

1. 用户上传 `.md` / `.txt`。
2. 后端按段落和长度切成 chunk。
3. Spring AI 调 OpenAI `text-embedding-3-small` 生成 1536 维向量。
4. pgvector 存储 chunk 内容、metadata 和 embedding。
5. 用户提问时先向量召回 topK。
6. 将召回内容作为上下文交给 `gpt-5-mini` 生成答案。

## 3. 工具调用链路

`OrderTools.queryOrder(orderNo)` 是一个本地模拟业务工具，覆盖订单、物流、支付和退款状态。订单类问题让模型调用工具，工具调用记录会回传到前端。

可演示订单号：

- `ORD-2026-0001`：已发货
- `ORD-2026-0002`：待财务确认
- `ORD-2026-0003`：退款处理中

## 4. 三个标准问题

### 命中文档

问题：`企业退款规则是什么？`

预期：返回退款规则，并展示 `company_knowledge.md` 的来源片段。

### 文档没有答案

问题：`知识库里有没有提到加班餐补标准？`

预期：说明知识库未提供足够依据，避免编造。

### 触发工具

问题：`查询 ORD-2026-0001 的订单状态`

预期：触发 `queryOrder`，返回已发货、预计送达时间和物流事件。

## 5. 可扩展点

- 文件解析：接入 PDF、Word、Excel 解析。
- 权限：按用户/部门过滤文档 metadata。
- 召回：加入 hybrid search、rerank、多路召回。
- 观测：记录 token、耗时、召回命中率、工具调用成功率。
- 生产化：异步入库、文件去重、失败重试、审计日志。
